# Reports

This folder documents the public interpretation boundary, validation approach, and data dictionary for the project outputs.

## Files

### VALIDATION_SUMMARY.md

Summarizes the validation-first workflow and the current accepted public boundary.

### DATA_DICTIONARY.md

Defines the public output files and common fields.

### PUBLIC_RESULTS_BOUNDARY.md

Defines what the public outputs can and cannot support.

## Review order

Recommended reading order:

1. PUBLIC_RESULTS_BOUNDARY.md
2. VALIDATION_SUMMARY.md
3. DATA_DICTIONARY.md

## Interpretation boundary

The reports preserve a conservative source boundary. Public outputs should be treated as strict EIA-860M classifier outputs, not final market-total estimates.
